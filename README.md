# Тестовые задания

Репозиторий с решениями трёх заданий: анализ требований, проектирование REST API и проектирование микросервисной архитектуры.

## Задание 1: анализ требований

Анализ ТЗ на функциональность корзины: найдены противоречия, неполные формулировки и формальный недочёт. Предложены варианты исправления, уточняющие вопросы заказчику и итоговая исправленная версия ТЗ.

Полный текст анализа — в файле `Задание_1.docx`.

## Задание 2: проектирование API

Для экрана "Выберите магазин" (интернет-магазин "Петрушка Зелёная") описаны:
- пример REST API запроса, вызываемого при переходе на экран;
- пример ответа в формате JSON, соответствующий макету, включая ссылку на внешний ресурс для перехода по клику на карточку магазина.

Полный текст — в файле `Задание_2.docx`.

## Задание 3: архитектура

Схема описывает поток обработки события "товар добавлен в корзину" через delay queue с таймером 24 часа, шину событий, сервис уведомлений (с проверкой токенов и согласий через сервис пользователей), хранилище шаблонов, дедупликацию и троттлинг, очередь push-задач и push-диспетчер с выбором провайдера (FCM для Android, APNs для iOS), завершаясь логированием статуса отправки и показом push-уведомления на устройстве пользователя.

Схема — в файле `микросервисная_архитектура.jpg`.
